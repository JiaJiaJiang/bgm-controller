# bgm-controller

这是一个通过纯前端网页使用"Web Audio API"实现的背景音乐播放控制器，适配手机和电脑浏览器。

使用AI deepseek-v4-flash 编写和人工排除bug。

使用vue3和vite构建。

支持导入导出json格式配置文件，并且所做配置会直接自动保存在localStorage中。（json直接利用vue的option api形式的数据对象存储特性导出和覆盖设置数据）

## 使用

这是一个静态页面项目，你可以把dist目录中的所有内容拷贝到任何web服务中加载即可。

如果需要启动开发服务器，需要在执行`npm install`之后执行`npm run dev`，然后根据控制台中显示的端口访问这个项目页面。

在线示例：[https://jiajiajiang.github.io/bgm-controller/dist/](https://jiajiajiang.github.io/bgm-controller/dist/)

## 音频源说明

场景中的音频源需要填入有效的音频绝对地址，比如 "https://luojia.me/a.mp3" ，或者如果你把音频放在了项目的"public/audios"目录中（开发模式启动的vite服务）或者"dist/audios"目录中（直接使用dist中的静态页面），那么可以填写相对地址"/audios/a.mp3"即可加载音频。

## 场景条目特性

- 场景名称
- 场景音量（音量百分比以总体音量为基础）
- 场景顺序（决定列表排序和切换顺序，必须唯一，重复自动后移）
- 淡入时间(切换音频时前后两首歌的叠加渐变时间，默认0.1s)
- 切换模式（顺序单次|顺序循环|随机循环|单曲循环|单曲播放）
- 音频源地址列表
  - 允许设置片段范围的时间（只播放此音频的指定片段）
  - 允许单独设置每首歌的音量（音量百分比以场景音量为基础）
  - 独立的试听播放按钮

## 快捷键

| 按键 | 功能 |
|------|------|
| 空格 | 播放 / 暂停 |
| [ | 上一个音频 |
| ] | 下一个音频 |

## 逻辑细节

- 预加载所有音频源，使用 `new Audio()` + `createMediaElementSource` 实现跨域音频加载
- 每个场景的每个音频独立创建 Audio 对象和 MediaElementSourceNode，永久连接到场景的 GainNode
- 所有场景 GainNode → 主 GainNode（总音量）→ destination，连接永久固定，不需要断开
- 播放控制只通过 `audio.play()` / `audio.pause()`，不涉及节点连接操作
- 场景切换时使用 JS interval 实现交叉淡入淡出（旧场景 fadeGain 淡出到 0，新场景 fadeGain 从 0 淡入到 1）
- 淡入淡出完全由 JavaScript 控制，不依赖 Web Audio API 的自动化调度，避免竞态问题
- 暂停时停止淡入淡出处理，节省计算资源
- url中可带config参数用于自动加载在线json配置文件，如果没有指定config文件，则尝试从localStorage中读取配置文件，都没有就所有数据默认留空
- 点击新建配置按钮必须二次确认才可以重置当前数据
- 音频加载失败时会弹出提示，显示失败的音频地址、所属场景、错误原因和处理建议
- 加载失败的音频在场景编辑器中显示红色标记，切换音频时自动跳过
- 场景编号必须唯一，重复时自动后移
